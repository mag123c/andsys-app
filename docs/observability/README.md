# Observability (모니터링 & 분석)

4ndSYS 앱의 모니터링, 분석, 알림 시스템 기획 문서.

## 기획 개요

### 목표

```
"측정할 수 없으면 개선할 수 없다. 하지만 모든 것을 측정하면 아무것도 개선하지 못한다."
```

1. **에러 감지**: 프로덕션 에러 발생 시 실시간 알림
2. **사용자 유입 분석**: SEO 효과, SNS 홍보 성과 측정
3. **제품 사용 패턴**: 핵심 기능 사용률, 퍼널, 리텐션
4. **성능 모니터링**: 페이지 로딩 속도, Core Web Vitals

### 데이터 흐름

```
사용자 유입 (GA4) → 제품 사용 (PostHog) → 성능 (Vercel) → 에러 (Sentry)
     ↓                    ↓                    ↓              ↓
  마케팅 효과         기능 개선            성능 최적화      버그 수정
```

### 선정 도구

| 도구 | 역할 | 비용 | 한도 |
|------|------|------|------|
| **Sentry** | 에러 트래킹 + 알림 허브 | 무료 | 5K errors/월 |
| **GA4** | 유입/마케팅 분석 | 무료 | 무제한 |
| **PostHog** | 제품 분석 | 무료 | 1M events/월 |
| **Vercel Analytics** | 성능 분석 | 무료 | 50K events/월 |

**월 비용: $0**

### 알림 체계

```
Sentry 에러 발생 → Slack/Discord/Email 즉시 알림
                         ↓
                    개발자가 확인
```

- 매번 대시보드 접속 불필요
- 문제 발생 시에만 알림 수신

## Quick Start

### 우선순위

```
1순위: Sentry (에러 + 알림) - 프로덕션 필수
2순위: GA4 (유입 분석) - SEO/마케팅
3순위: PostHog (제품 분석) - 기능 개선
4순위: Vercel Analytics (성능) - 최적화
```

### 예상 소요

```
Sentry 설정      : 코드 + 알림 설정
GA4 설정         : 스크립트 삽입
PostHog 설정     : 스크립트 + 커스텀 이벤트
Vercel Analytics : 대시보드에서 활성화
```

## 핵심 원칙

1. **무료 우선**: MVP 단계에서 비용 $0 유지

3. **실시간 알림**: 대시보드 접속 없이 문제 파악
4. **추상화 레이어**: 나중에 도구 교체 용이하게 설계
