# 디자인 시스템 & UX 가이드

## 디자인 원칙

- **Minimal**: 글쓰기 집중, 불필요한 장식 배제
- **Calm**: 차분한 색상, 부드러운 전환
- **Consistent**: 동일한 패턴과 언어
- **Accessible**: 다크/라이트 모드, 키보드 네비게이션

## 스타일 시스템

### 기본 설정
- **테마**: shadcn/ui 기본 테마 사용
- **폰트**: Geist (sans), Geist Mono (code)
- **아이콘**: Lucide React
- **스페이싱**: Tailwind 4px 기반 (gap-1 ~ gap-8)

### 컴포넌트 계층
| 버튼 | 용도 |
|------|------|
| default | 주요 액션 |
| secondary | 보조 액션 |
| ghost | 툴바, 아이콘 버튼 |
| destructive | 삭제 (확인 필수) |

## UX 패턴

### 상태 표시

| 상태 | 패턴 |
|------|------|
| 로딩 | 버튼: disabled + Loader2, 콘텐츠: Skeleton |
| 에러 | 폼: 인라인 메시지, 네트워크: Toast + 재시도 |
| 빈 상태 | 아이콘 + 설명 + 액션 버튼 |
| 저장 | "저장 중..." → "저장됨" → "오프라인" |

### 피드백

| 상황 | 방식 |
|------|------|
| 성공 | Toast (자동 닫힘) |
| 경고 | Toast (수동 닫힘) |
| 확인 필요 | Dialog |
